Ten aspects of comparison between React JSX & HTML

1. Specifying the type of element in HTML & React

In HTML:Specifying the type of HTML element

For ex: <h1> Hello world />

Here the type of h1 is an object & we usually call them as an element in html.

To ensure the type of HTML element, console them as below.

let h1 = document.getElementsByTagName('h1');

console.log(typeof(h1)); // is an object

Note: An HTML element is case-insensive but during development it is preferred to use lowercase tag elements.

In React:Specifying the React element type

The first part of the tag species the type of the react element

ex : <div className="Icon-wrapper"/>

Though div tag comes under the HTML family, but in a react application we call them as react element.

Note: Component Should be Started with Capital letter, indicating react, it's a React Component.

In React Jsx: Components are the building blocks of an application. It can be a class or a functional component

Here the type of component can be a class or a function

4. Attributes in HTMl vs Props in a Component

In Html: We have Attributes attached to each DOM element, they follow the structure as below

<input type="text" name="name" id="firstname" placeholder="Enter Your Name">

Here, type, name, id, placeholder are called as attributes in HTML, we can assume these attributes as properties of the corresponsing DOM Element.Also,there are four types of attributes in html, they are required attributes,optional attributes, standard attributes, Event attributes

In most of the cases, the attribute type in HTML is a string.

They are pinned to the DOM as: input = { type: "text", name: "name", id: "firstname", placeholder: "Enter Your Name"}

In React:In a component, instead of attributes, we call them with a name called as 'props' whose type is an object

Here, props are passed to the Input component as object, where each of them are the key-value pairs of the props object

In React: The props list can be a string type, number type, boolean type or else we can even have objects & functions.

5. Insertion scenario of Elements in HTML vs React

In Html:There is no scenario of any function calls behind the scenes while you're writing any HTML element & rendered them into the DOM. Because those element are all supported by all the browsers.

In React:Since in React, it is possible to write Javascript along with HTML(JSX), so for each element before insertion into the DOM React.createElement() function call happen for conversion of JSX to pure JS.

Note: During conversion of JSX into pure js, 'React' must be in scope because jsx calls to React.CreateElement(); so we need to import React in the top level.

6. Structure of a Individual Element in HTML vs React

In HTML:An Individual HTMl element starts with a tag-name(type is an object), followed by the attributes of that element. The attribute value of an HTML element in most of the cases was "String" type.

The value of the attribute enclosed in a double quotes can be of:

1. String type assigned to the attribute (E.g: < input type="text" >) - here "type" attribute's value is a string.

2. Path (absolute/relative) (E.g: < img src="/images/blogpost.jpg" />) - here the value of src enclosed in a double quotes is a path of that image.

3. Attribute without a value(E.g: < input type="text" required />)- here the required attribute without a value, meant for HMTL5 Validation

A Typical HTML element looks like:-

<Object attribute1='value' attribute2='value' attribute3='value'> content... <Object/>

In React:An Individual React element starts with a Component name(component can either be a function or class), followed by the props of that element.

The value of the prop passed to the component can be:

1. String Variable

2. Boolean Variable

3. Number Variable

4. Object

5. Function

6. May be a state variable of the component

A Typical React element looks like:-

<Component prop1={value} prop2={value} prop3={value} />

7. Handling Scripts in HTML vs React

In HTML:Two possible ways of writing scripts for an html document - internal sscripts(using script tag) & external scripts(added via <link> tag)

In React:A Component file itself like .js (or) .jsx, where the scripts are placed above return() of a component & react elements are placed inside the return of a component

The advantage JSX is we could able to place & write expressions,calling the functions even including anonymous functions along with react elements

8. Concept of Reusability in HTML vs React

In HTML: The concept of reusability of an HTML tag is nowhere. we are tend to add those tag everytime whereever necessary.

In React: we can able to reuse the Input Component by passing the necessary props, while inspecting the DOM, we could able to find them as a single HTML element, but actually we are keeping a copy of that component by simply passing the props.

Theme behind it was, for a component to be reusable, it has to take in data or data sets (via props) and return an output (usually through a function passed via props).

For E.g: In a input component, We can achieve different type of React Element by simple passing the prop "type"where we are consuming the component.

Advantages: Saves time by ensuring: -

1. Less code is written

2. Development is faster

3. Maintenance is easy

9. Handling of Reserved Keywords of Javascript in HTML vs React

In HTML:The Separation of the keywords is done through'Script' tags for internal scripts and external scripts attached through 'link' tags.

In React:Since it possible to write Scripts & tag elements in a single file, in order to avoid clash of Javascript reserved keywords in JSX, we have new reserved keywords for JSX

E.g: 1. We have reserved keywords in JSX like instead of 'for' use 'htmlFor' because compiler asssumes 'for' as a keyword for looping.

E.g: 2. Reserved keywords in JSX like instead of 'class' we use 'className'.

10. Storing of elements in HTML vs React

In HTML:We can able to catch an element using a WebAPI like 'getElementsByTagName'. Then we can manipulate the element but it is not possible to store an element directly as in JSX.
